Bug Title
    Swagger defines username as required but API creates employee without it

Environment
    Environment: Prod
    Endpoint: POST /api/Employees
    Date discovered: 02/27/2026

Severity
    medium

Priority
    medium

Steps to Reproduce
    1. Include valid Authorization in request
    2. Send a post Request with the following payload without username
    {
    "firstName": "Luis",
    "lastName": "Ramos",
    "dependants": 2
    }

Expected Result
    - Status Code: 400 Bad Request
    - Appropriate error message indicating missing username field

Actual Result
    Status Code: 200 OK
    Employee created successfully

Impact
    Contract violation, the Swagger defines user as an required field in the payload
    this shows inconsistent contract with implementation
